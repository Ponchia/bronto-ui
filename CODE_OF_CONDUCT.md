# Code of Conduct

This project follows a simple rule: keep technical collaboration
respectful, specific, and safe for public participation.

## Expected behaviour

- Be direct about technical risks while staying respectful toward people.
- Assume good intent, especially when reviewing first-time contributions.
- Keep disagreement focused on reproducible behaviour, documented
  trade-offs, and maintainable code.
- Keep credentials, private infrastructure details, and personal data
  out of issues, PRs, and discussions.

## Unacceptable behaviour

- Harassment, personal attacks, or discriminatory language.
- Publishing others' private information.
- Sustained disruption of technical discussion.

## Scope

Applies to all project spaces — issues, pull requests, discussions, and
any other channel that represents the project.

## Reporting

This is a solo-maintained project. Report conduct concerns privately via
GitHub's private vulnerability reporting (see [SECURITY.md](SECURITY.md))
if the report includes sensitive information; otherwise open an issue.
Reports are handled in good faith and as confidentially as the channel
allows.
