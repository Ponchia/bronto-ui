# CSS quality & maintainability — bronto-ui review
**Verdict:** Mature CSS architecture: the default surface is layered deliberately (`css/core.css:10`), dist is byte-checked as a single `@layer bronto` build (`scripts/check-dist.mjs:70`), and token/drift gates are unusually strong (`scripts/lib/artifacts.mjs:31`). The main long-term risks are not craft failures, but guardrail gaps around duplicate CSS, per-leaf custom-property dependencies, and intentionally global base/reset behavior (`css/base.css:5`, `knip.json:5`).
**Grade:** A- — strong architecture and gates, with maintainability gaps in duplication and leaf self-containment.
## Strengths
- Single-layer cascade contract is clear: `core.css` declares `@layer bronto` once and imports every default leaf with `layer(bronto)` (`css/core.css:10`, `css/core.css:12`).
- Dist preserves the contract: bundles concatenate leaves and wrap them in one `@layer bronto` (`scripts/build-dist.mjs:53`), and `check-dist` rejects stale files, nested layers, and budget/url drift (`scripts/check-dist.mjs:60`, `scripts/check-dist.mjs:70`, `scripts/check-dist.mjs:79`).
- Layered and raw escape-hatch exports are mechanically checked: `./css/<leaf>` must target `dist/css`, while `./css/unlayered/<leaf>` must target raw source (`scripts/check-exports.mjs:77`, `scripts/check-exports.mjs:100`).
- Token source-of-truth is well designed: `tokens/index.js` owns `cssVars` and says `css/tokens.css` palette blocks are generated from it (`tokens/index.js:7`); the hand-authored tail is preserved below a marker (`css/tokens.css:230`, `scripts/gen-tokens-css.mjs:120`).
- Custom-property typo protection exists: `check-variables` collects defined CSS properties and flags no-fallback unresolved `var()` references (`scripts/check-variables.mjs:39`, `scripts/check-variables.mjs:50`).
- Modern CSS is used intentionally, not casually: the browser floor explicitly includes layers, `:has()`, `color-mix()`, `@starting-style`, OKLCH, and no legacy fallbacks (`CONTRIBUTING.md:131`); fragile features are gated, e.g. `::details-content` (`css/disclosure.css:149`) and reduced-motion view transitions (`css/motion.css:381`).
- File cohesion is strong: opt-in leaves document their boundary and host-owned responsibilities, e.g. generated trust surfaces (`css/generated.css:4`), report grammar (`css/report.css:4`), and spark geometry (`css/spark.css:8`).
## Weaknesses / risks
- [P2] Duplicate CSS is not guarded: Knip explicitly disables duplicate detection (`knip.json:5`), while Stylelint disables `no-descending-specificity` (`.stylelintrc.json:9`). Overlapping rules can accrete silently as the 46-leaf surface grows.
- [P2] Per-leaf variable dependency is not fully checked: `check-variables` treats a var as valid if defined anywhere in `css/` (`scripts/check-variables.mjs:39`), but `legend.css` gradients require `--chart-seq-*` with no fallback (`css/legend.css:149`), so importing legend without `dataviz.css` can invalidate gradient paint.
- [P2] Global reset/base styles are intentionally host-invasive: `base.css` styles `*`, `html`, `body`, `a`, `p/li`, headings, and form controls globally (`css/base.css:5`, `css/base.css:11`, `css/base.css:47`, `css/base.css:71`, `css/base.css:78`). Layers make overrides easier, but the root import is not neutral in an embedded host app.
- [P2] Only `--value` is typed with `@property` (`css/feedback.css:13`), while many public numeric geometry knobs remain untyped, e.g. `--lo/--hi/--v` (`css/interval.css:6`), bullet `--v/--t/--b1/--b2` (`css/bullet.css:12`), and spotlight coordinates (`css/spotlight.css:15`). Bad units can still poison `calc()` geometry.
- [P3] `css/generated.css` is hand-authored despite its name: it describes AI/generated-content trust UI and is not a generated artifact (`css/generated.css:1`, `scripts/lib/artifacts.mjs:31`). The name is semantically correct but easy to misread in maintenance.
- [P3] A stale comment exists in color-policy enforcement: only `legend.css` is exempted for chart-token consumption (`scripts/check-color-policy.mjs:159`), but the later comment says `report.css` chart parts are allowed (`scripts/check-color-policy.mjs:381`).
## Top recommendations
1. Add a CSS duplication/overlap gate, or enable Knip duplicate detection for CSS, because duplicate detection is currently off (`knip.json:5`) and specificity ordering is intentionally not linted (`.stylelintrc.json:9`).
2. Extend `check-variables` to understand per-leaf prerequisites or require fallbacks for optional-token reads; start with `legend.css` chart ramps (`css/legend.css:149`, `scripts/check-variables.mjs:50`).
3. Register high-risk numeric knobs with `@property`, following the proven `--value` pattern (`css/feedback.css:7`, `css/feedback.css:13`), especially interval, bullet, spark, and spotlight geometry props (`css/interval.css:34`, `css/bullet.css:42`, `css/spark.css:27`, `css/spotlight.css:31`).
4. Keep raw unlayered exports, but document them as dangerous escape hatches wherever direct leaf imports are taught; the export map makes them easy to consume accidentally (`package.json:297`, `package.json:337`).
5. Add a short “not generated artifact” banner to `css/generated.css`, since true generated leaves already use explicit generated banners (`css/dataviz.css:1`, `css/skins.css:1`, `css/generated.css:1`).
6. Fix the stale `report.css` comment in `check-color-policy` so future maintainers do not widen the chart-token exemption incorrectly (`scripts/check-color-policy.mjs:381`, `scripts/check-color-policy.mjs:386`).
## Notable observations
- No real `#id` selector risk surfaced; the opened ID-related rule is attribute-based scroll-margin only (`css/base.css:38`).
- `!important` use is concentrated in accessibility/print escape hatches, with rationale inline for reduced motion (`css/motion.css:396`) and print citation suppression (`css/sources.css:231`).
- `dist/bronto.css` is the actual root export (`package.json:242`) and is generated/minified into a single line beginning with `@layer bronto` (`dist/bronto.css:1`).
