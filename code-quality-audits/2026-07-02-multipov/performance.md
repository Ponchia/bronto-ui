# Performance & bundle — bronto-ui review
**Verdict:** The default CSS payload discipline is real: `check-dist` rebuilds generated CSS, byte-compares it, enforces one `@layer bronto`, validates `url()` assets, and applies raw/gzip budgets (`scripts/check-dist.mjs:20`, `scripts/check-dist.mjs:66`, `scripts/check-dist.mjs:70`, `scripts/check-dist.mjs:75`). The README headline is honest for the default identity bundle, not for all shipped optional surfaces: `bronto.css` is the root export (`package.json:242`) and opt-in analytical/report leaves are explicitly excluded from that bundle (`README.md:88`). The main gaps are aggregate/package-size visibility and JS import granularity.
**Grade:** B — strong CSS gates, weaker per-layer budgets and behavior/adaptor granularity (`scripts/build-dist.mjs:179`, `package.json:406`).

## Strengths
- Root import is CSS-only and points at `dist/bronto.css`, with CSS marked as the only side-effect class for bundlers (`package.json:41`, `package.json:114`, `package.json:242`).
- `check-dist` is a real gate: it rebuilds bundles in memory, rejects stale committed `dist`, enforces one `@layer bronto`, checks raw/gzip budgets, and validates relative assets (`scripts/check-dist.mjs:20`, `scripts/check-dist.mjs:66`, `scripts/check-dist.mjs:70`, `scripts/check-dist.mjs:75`, `scripts/check-dist.mjs:83`).
- CI and release both run `npm run check`, and `check` includes `check:dist` (`package.json:166`, `package.json:191`, `.github/workflows/ci.yml:115`, `.github/workflows/ci.yml:118`, `.github/workflows/release.yml:78`, `.github/workflows/release.yml:80`).
- Default identity CSS stays separate from analytical/report layers: `core.css` imports only core identity leaves, while `analytical.css` and `report-kit.css` are separate roll-ups (`css/core.css:12`, `css/core.css:26`, `css/analytical.css:15`, `css/report-kit.css:14`).
- CSS opt-in paths are granular and enforced: each leaf has layered and unlayered export targets, and the export gate checks that shape (`package.json:252`, `package.json:297`, `scripts/check-exports.mjs:77`, `scripts/check-exports.mjs:100`).
- Zero runtime dependency policy is machine-checked against `dependencies`, optional/bundled dependency fields, and optional peer metadata (`scripts/check-exports.mjs:157`, `scripts/check-exports.mjs:176`, `package.json:226`, `package.json:231`).
- Behavior modules are on-demand and SSR-safe at call time: initializers guard on DOM availability and return cleanup/no-op paths (`behaviors/internal.js:19`, `behaviors/theme.js:52`, `behaviors/table.js:37`, `scripts/check-consumer-surface.mjs:177`, `scripts/check-consumer-surface.mjs:219`).
- Font loading avoids FOIT with `font-display: swap`, uses woff2 only, and has a reduced-data opt-out path for Doto usage (`css/fonts.css:9`, `css/fonts.css:18`, `css/fonts.css:22`, `css/fonts.css:70`).

## Weaknesses / risks
- [P1] Behaviors/adapters are not publicly granular by leaf: package exports only `./behaviors` and whole framework adapter subpaths, while the barrels re-export/import every behavior. What breaks: no-bundler consumers cannot take only `initDialog` through a stable package export; they pay parse/instantiate cost for the behavior surface and adapters depend on bundler tree-shaking (`package.json:406`, `behaviors/index.js:33`, `behaviors/index.js:54`, `react/index.js:48`, `react/index.js:73`).
- [P2] The hard CSS budget is one shared ceiling for every emitted file, not per surface or aggregate. What breaks: `report-kit`, `analytical`, total `dist/css`, fonts, and tarball size can grow without a tailored budget as long as each file stays below the default bundle ceiling (`scripts/check-dist.mjs:75`, `scripts/build-dist.mjs:117`, `scripts/build-dist.mjs:124`, `scripts/build-dist.mjs:179`, `scripts/size-report.mjs:2`).
- [P2] The README badge/headline can be misread as total package cost. What breaks: consumers may miss that the npm package intentionally ships source CSS, generated dist CSS, fonts, JS surfaces, and docs in addition to the default bundle (`README.md:6`, `README.md:75`, `README.md:88`, `package.json:44`, `package.json:113`).
- [P2] Some default motion still animates layout/paint properties: shimmer animates `background-position`, indeterminate progress animates `inset-inline-start`, and determinate bars transition `inline-size`. What breaks: many skeletons/progress bars can create unnecessary paint/layout work versus transform-based animation (`css/motion.css:112`, `css/motion.css:132`, `css/feedback.css:427`, `css/feedback.css:431`, `css/feedback.css:487`).
- [P2] Root-level `html:has(...)` scroll lock is broad. What breaks: modal open/close can invalidate style matching from the document root; acceptable on the stated evergreen floor, but it is the most expensive selector shape in core (`css/overlay.css:91`, `css/overlay.css:95`).
- [P2] Opt-in spotlight uses a huge `box-shadow` cutout and transitions `inline-size`/`block-size`. What breaks: guided-tour overlays can repaint and relayout during target moves, especially on large viewports (`css/spotlight.css:29`, `css/spotlight.css:33`, `css/spotlight.css:42`).
- [P3] Fonts are discovered through CSS rather than a documented preload path. What breaks: no FOIT, but first Doto use can FOUT and wait for CSS/font discovery; core imports `fonts.css` by default (`css/core.css:12`, `css/core.css:13`, `css/fonts.css:18`, `css/fonts.css:64`, `README.md:75`).
- [P3] `size-report` is informational and suppresses `npm pack` stderr. What breaks: publish-preflight/debug output can hide the real pack failure reason until `npm pack` is run directly (`scripts/size-report.mjs:2`, `scripts/size-report.mjs:48`, `scripts/size-report.mjs:53`, `scripts/lib/shipped-files.mjs:26`, `.github/workflows/release.yml:124`).

## Top recommendations
1. Add explicit budgets for `dist/bronto.css`, `dist/css/analytical.css`, `dist/css/report-kit.css`, largest leaf, total generated CSS, `fonts/*`, and packed tarball; keep `check-dist` as the hard default-bundle gate (`scripts/check-dist.mjs:75`, `scripts/build-dist.mjs:179`, `scripts/size-report.mjs:43`).
2. Rename the badge/quick-start wording to “default CSS bundle ~88 KiB / ~15 KiB gzip” and add a one-line “not total package size” note beside the opt-in-layer caveat (`README.md:6`, `README.md:75`, `README.md:88`, `package.json:44`).
3. Export stable behavior leaf subpaths such as `./behaviors/dialog` and consider generated per-hook adapter subpaths for React/Solid/Svelte/Vue so non-bundled consumers can import one behavior without the barrel (`package.json:406`, `behaviors/index.js:33`, `react/index.js:48`, `svelte/index.js:36`).
4. Convert progress/meter fill animation to transform-based scale/translate where possible, and keep `inline-size` transitions only where the layout effect is intentional (`css/motion.css:132`, `css/feedback.css:423`, `css/feedback.css:427`, `css/feedback.css:483`, `css/feedback.css:487`).
5. Add documented font-loading options: preload snippets for Doto users and a first-class no-font/source-leaf recipe for consumers overriding `--display` / `--dot-font` (`css/fonts.css:13`, `css/fonts.css:18`, `css/fonts.css:66`, `README.md:86`).
6. Add a CSS performance allowlist check for layout-property animations, root `:has()`, and large repaint effects, with current exceptions documented in code (`css/overlay.css:95`, `css/spotlight.css:33`, `css/spotlight.css:42`, `css/motion.css:112`).

## Notable observations
- `css/generated.css` is not a generated artifact; it is an opt-in “generated content” trust-surface stylesheet, despite the name (`css/generated.css:1`, `css/generated.css:9`).
- The font URL rewrite for layered leaves is deliberate: source leaves use `../fonts`, generated `dist/css/*` leaves are rewritten to `../../fonts`, and `check-dist` validates those URLs (`scripts/build-dist.mjs:64`, `scripts/build-dist.mjs:70`, `scripts/check-dist.mjs:79`, `dist/css/fonts.css:1`).
- `skins.css` and `dataviz.css` are generated and explicitly opt-in, so colorways/chart palettes do not burden the default identity bundle (`css/skins.css:1`, `css/skins.css:4`, `css/dataviz.css:1`, `css/dataviz.css:4`).
- Reduced-motion handling is unusually thorough: view-transition pseudo-elements are reset separately from the universal animation/transition reset (`css/motion.css:381`, `css/motion.css:399`).
- The package intentionally ships curated docs and both CSS source/dist surfaces; `check-pack` treats that as deliberate package surface, not accidental leakage (`scripts/check-pack.mjs:44`, `scripts/check-pack.mjs:74`, `scripts/check-pack.mjs:80`).
