# Architecture & design-system coherence — bronto-ui review
**Verdict:** The architecture is broadly sound and unusually well-policed: core CSS is intentionally layered, token mirrors are generated, opt-in report/tooling leaves stay outside the default bundle, and primitive JS mostly stops at generic ARIA/geometry kernels (`css/core.css:6`, `scripts/check-dist.mjs:70`, `docs/architecture.md:114`, `docs/frontier-primitives.md:18`). The main coherence gaps are claim-level: “un-layered CSS wins with no `!important`” has deliberate exceptions, and “live accent / re-skins for free” is not true for resolved foreign-renderer outputs (`css/motion.css:396`, `docs/vega.md:146`, `tokens/vega.js:14`).
**Grade:** B+ — strong system thesis and gates, with a few thesis-visible edge cases.

## Strengths
- The single-layer strategy is real for the default and published leaf surfaces: `core.css` declares/populates `@layer bronto`, direct leaves are emitted as one `@layer bronto{...}` block, and `check-dist` rejects nested/stray layer rules (`css/core.css:10`, `css/core.css:12`, `scripts/build-dist.mjs:70`, `scripts/check-dist.mjs:70`).
- The package contract cleanly distinguishes safe layered imports from explicit raw escape hatches, so `./css/foo.css` maps to `dist/css/foo.css` while `./css/unlayered/foo.css` maps to authored source (`docs/package-contract.md:16`, `docs/package-contract.md:79`, `package.json:252`, `package.json:297`).
- Core token values have a clear source and projection pipeline: `tokens/index.js` owns `cssVars`, `gen-tokens-css` emits the CSS palette blocks, and the artifact registry drift-checks `tokens.css`, DTCG, resolved JSON, Figma variables, and token JSON together (`tokens/index.js:7`, `scripts/gen-tokens-css.mjs:2`, `scripts/lib/artifacts.mjs:31`, `scripts/lib/artifacts.mjs:38`).
- The resolved-token story is honest for non-CSS renderers: `gen-resolved` evaluates `var()` plus sRGB/OKLCH `color-mix()`, while DTCG marks CSS-runtime-derived values as `$value: null` with a CSS extension instead of inventing static values (`scripts/gen-resolved.mjs:6`, `scripts/gen-resolved.mjs:149`, `scripts/gen-dtcg.mjs:9`, `scripts/gen-dtcg.mjs:56`).
- Primitive boundaries mostly hold in implementation: spark and interval require host-normalized values, crosshair emits pointer fractions but not data values, command emits selection events but does not execute actions, and legend toggles control state without hiding chart series (`css/spark.css:8`, `css/interval.css:6`, `behaviors/crosshair.js:19`, `behaviors/command.js:20`, `behaviors/legend.js:19`).
- Identity vs opt-in toolbox separation is enforced by import topology: core imports app/forms/tables/feedback but no report/tooling leaves, `analytical.css` imports only analytical primitives, and `report-kit.css` explicitly excludes workbench/command surfaces (`css/core.css:12`, `css/core.css:26`, `css/analytical.css:15`, `css/report-kit.css:9`).

## Weaknesses / risks
- [P1] Foreign-renderer “live accent” claims are misleading: Vega docs say category 1 “re-skin[s] for free with `--accent`,” but generated Vega/Mermaid/charts artifacts contain fixed red hexes and `emit-theme` accepts only `light|dark`, so skins/custom accents do not propagate to canvas/SVG renderer configs without manual regeneration/override (`docs/vega.md:146`, `tokens/vega.js:14`, `tokens/mermaid.js:37`, `tokens/charts.json:2`, `scripts/emit-theme.mjs:45`).
- [P2] The “un-layered CSS wins with no `!important`” claim is true for normal declarations but not for deliberate layered important rules; reduced-motion and print resets use `!important`, so consumers may need equal/greater importance for those cases despite the headline claim (`css/core.css:6`, `css/core.css:8`, `css/motion.css:396`, `css/motion.css:402`, `css/base.css:270`).
- [P2] Token “single source” is actually a tiered source model: core tokens live in `tokens/index.js`, skins in `tokens/skins.js`, charts in `tokens/charts.js`, and renderer slot maps in generator scripts; the model is workable, but future semantic changes can miss bridge maps because those mappings are not token data (`tokens/index.js:18`, `tokens/skins.js:24`, `tokens/charts.js:4`, `scripts/gen-mermaid.mjs:36`, `scripts/gen-vega.mjs:60`).
- [P2] Print palette values are duplicated between global token print remapping and report-specific print remapping, creating a drift point for the “token values come from one place” thesis (`css/tokens.css:327`, `css/tokens.css:349`, `css/report.css:686`, `css/report.css:693`).
- [P3] The chart-token leakage gate has a comment/implementation mismatch: the comment says report chart helpers may consume chart tokens, but only `legend.css` is actually exempt, so future report chart affordances would fail contrary to the comment (`scripts/check-color-policy.mjs:159`, `scripts/check-color-policy.mjs:163`, `scripts/check-color-policy.mjs:381`).
- [P3] Primitive/app boundaries carry some compatibility coupling: primitive rules permanently group app-shell aliases like `.ui-app-metrics`, `.ui-app-metric`, and `.ui-app-empty-state`, which keeps output stable but makes primitives less shell-agnostic than the mental model suggests (`css/primitives.css:163`, `css/primitives.css:168`, `css/primitives.css:759`, `css/app.css:257`).

## Top recommendations
1. Add skin/custom-accent support or explicit caveats for resolved renderer bridges: either generate `{theme, skin}` maps from `tokens/skins.js`, accept an accent override in Vega/Mermaid/chart helpers, or remove “re-skins for free” from resolved-output docs (`tokens/skins.js:37`, `scripts/emit-theme.mjs:45`, `docs/mermaid.md:81`).
2. Qualify the cascade promise: document that unlayered normal CSS beats layered normal CSS, while accessibility/print/reduced-motion `!important` rules are intentional exceptions (`css/core.css:6`, `css/motion.css:396`, `css/base.css:273`).
3. Name the token architecture as “tiered sources with generated projections,” not one monolith: core `cssVars`, skins, charts, and renderer maps each have an owner and gate (`tokens/index.js:21`, `tokens/skins.js:37`, `tokens/charts.js:45`, `scripts/check-mermaid.mjs:4`).
4. Centralize or gate duplicated print palette remaps so `tokens.css` and `report.css` cannot diverge silently (`css/tokens.css:352`, `css/report.css:693`).
5. Fix the `check-color-policy` report-chart exemption comment or add a real, scoped exemption if report chart helpers are intended (`scripts/check-color-policy.mjs:159`, `scripts/check-color-policy.mjs:385`).
6. Track app-shell aliases in a deprecation/compatibility table so primitive files can remain conceptually shell-agnostic even while preserving old class names (`css/primitives.css:164`, `css/primitives.css:760`).

## Notable observations
- The layered/unlayered export split is stronger than most CSS libraries: the raw source leaves are not accidental leakage but a named public escape hatch (`docs/package-contract.md:17`, `package.json:297`).
- The renderer bridges correctly avoid importing/running Mermaid, D2, or Vega at runtime; they are config/data projections, matching the “tokens, not components” thesis (`scripts/gen-mermaid.mjs:16`, `scripts/gen-d2.mjs:12`, `scripts/gen-vega.mjs:18`).
- `report.css` explicitly refuses to become a chart engine and points consumers to Vega or token-themed inline SVG instead (`css/report.css:662`).
- The matrix gates make surface growth scalable: CSS leaves, public behaviors, and helper exports require docs/tests/ownership before shipping (`scripts/check-component-matrix.mjs:1`, `scripts/check-behavior-matrix.mjs:1`).
