# CSS organization, cohesion & duplication — bronto-ui structure review
**Verdict:** The package-level architecture is strong: `css/core.css` has a clear 15-leaf core bundle boundary, opt-in rollups are explicit, and `dist/` is generated/guarded rather than hand-maintained (`css/core.css:12`, `css/core.css:26`, `css/analytical.css:15`, `css/report-kit.css:14`, `scripts/check-dist.mjs:22`). The main risk is inside the leaves: large broad files and cross-file selector ownership mean maintainers must know implicit relationships across `base`, `primitives`, `content`, `report`, `disclosure`, `overlay`, and opt-in leaves to change one component safely (`css/base.css:204`, `css/content.css:107`, `css/disclosure.css:400`, `css/overlay.css:178`).
**Grade:** B- — good distribution contract, weaker long-term CSS cohesion.

## Strengths
- `core.css` is intentionally ordered and layered, with override semantics documented at the entrypoint (`css/core.css:6`, `css/core.css:9`, `css/core.css:12`, `css/core.css:26`).
- Opt-in aggregation is principled: analytical imports exactly figure/evidence leaves (`css/analytical.css:1`, `css/analytical.css:15`, `css/analytical.css:23`), while report-kit is a static/PDF report roll-up and explicitly excludes workbench/command surfaces (`css/report-kit.css:1`, `css/report-kit.css:9`, `css/report-kit.css:38`).
- `dist/` is generated from authored CSS: one flattened `dist/bronto.css`, one layered `dist/css/*.css` per leaf, and in-memory drift checking (`scripts/build-dist.mjs:53`, `scripts/build-dist.mjs:60`, `scripts/build-dist.mjs:117`, `scripts/check-dist.mjs:60`).
- Dead file risk is actively guarded: every authored `css/*.css` must be core, opt-in, or a roll-up, and stale `dist` CSS is rejected (`scripts/check-dist.mjs:22`, `scripts/check-dist.mjs:27`, `scripts/check-dist.mjs:38`, `scripts/check-dist.mjs:55`).
- Public surface accounting is unusually strong: docs say CSS selectors/class registry are aligned (`CONTRIBUTING.md:77`), package-contract records 46 layered CSS outputs (`docs/package-contract.md:312`), and the component matrix requires each shipped leaf to be foundation or component-owned (`scripts/check-component-matrix.mjs:1`, `scripts/check-component-matrix.mjs:384`).
- Many opt-in leaves are cohesive and self-bounded: `selection.css` states that it only owns selection emphasis, not brush logic (`css/selection.css:1`, `css/selection.css:4`, `css/selection.css:14`), and `highlights.css` only paints registered Custom Highlight ranges (`css/highlights.css:1`, `css/highlights.css:5`, `css/highlights.css:10`).

## Weaknesses / risks
- [P1] Several core leaves are “category buckets,” not single components: `primitives.css` spans layout, surfaces, metrics, buttons, links, badges, key/value, empty state, shortcuts, and timeline (`css/primitives.css:6`, `css/primitives.css:110`, `css/primitives.css:163`, `css/primitives.css:356`, `css/primitives.css:812`), while `disclosure.css` spans tabs, accordion, segmented control, breadcrumb, pagination, avatar, and carousel (`css/disclosure.css:8`, `css/disclosure.css:92`, `css/disclosure.css:178`, `css/disclosure.css:275`, `css/disclosure.css:400`). Cost: small changes require broad-file review and increase accidental regressions.
- [P1] Cross-file selector ownership is real: `base.css` owns forced-colors/print rules for progress, dots, switch, segmented, toast, app rail/topbar, site nav, theme toggle, modal, cards, tables, prose, and tabs (`css/base.css:204`, `css/base.css:224`, `css/base.css:259`, `css/base.css:283`, `css/base.css:292`). Cost: component behavior is split between the component leaf and global mode leaf.
- [P1] Table/prose/report responsibilities overlap: `table.css` owns `.ui-table` and `.ui-table-wrap` (`css/table.css:6`, `css/table.css:12`), `content.css` makes raw prose tables behave like UI tables (`css/content.css:250`, `css/content.css:260`), and `report.css` special-cases `.ui-table-wrap` inside report evidence (`css/report.css:367`, `css/report.css:371`). Cost: table rendering changes need three-file impact analysis.
- [P2] The `.ui-mark` boundary is fragile enough to need a specificity comment: core prose must avoid overriding opt-in marks (`css/content.css:107`, `css/content.css:112`), while `marks.css` owns `.ui-mark` styling (`css/marks.css:1`, `css/marks.css:14`). Cost: future prose selector edits can silently break the most documented opt-in mark use.
- [P2] Carousel ownership is split by composition: `disclosure.css` owns the scroll-snap carousel (`css/disclosure.css:400`, `css/disclosure.css:407`, `css/disclosure.css:436`), while `overlay.css` owns lightbox carousel constraints and image behavior (`css/overlay.css:178`, `css/overlay.css:203`, `css/overlay.css:211`). Cost: “carousel” is not fully local to one file.
- [P2] Repeated declaration blocks are structural, not incidental: the uppercase metadata label block repeats across prose captions, stat labels, report labels, compare heads, job metadata, and property labels (`css/content.css:242`, `css/primitives.css:201`, `css/report.css:231`, `css/report.css:614`, `css/state.css:149`, `css/workbench.css:157`). Cost: small typography changes require synchronized edits or drift.
- [P2] Link and disclosure animation patterns are duplicated: `.ui-prose a` and `.ui-link` share the same underline treatment (`css/content.css:92`, `css/primitives.css:491`), and accordion/tree duplicate the same `::details-content` transition pattern (`css/disclosure.css:164`, `css/tree.css:108`). Cost: fixes to one idiom may miss the parallel idiom.
- [P2] Current gates catch style, export drift, class registry drift, and dead package paths, but not CSS duplication/cohesion: stylelint is only `css/**/*.css`, knip is the dead-code check, and the full check chain has no duplicate-declaration/co-ownership gate (`package.json:116`, `package.json:148`, `package.json:150`, `package.json:191`). Cost: copy-paste can accumulate while CI stays green.
- [P3] Naming is mostly consistent `ui-*`, but some compressed vocabularies increase lookup cost: `selection.css` exports `.ui-sel*` instead of `.ui-selection*` (`css/selection.css:1`, `css/selection.css:10`, `css/selection.css:14`), and `sources.css` exports cross-cutting `.ui-src--*` trust states (`css/sources.css:11`, `css/sources.css:22`, `css/sources.css:28`). Cost: maintainers must remember abbreviations rather than infer names from file names.

## Top recommendations
1. Split the largest category buckets into internal authored partials while preserving public exports: start with `primitives.css` sections (`css/primitives.css:6`, `css/primitives.css:356`, `css/primitives.css:812`) and `disclosure.css` sections (`css/disclosure.css:8`, `css/disclosure.css:178`, `css/disclosure.css:400`).
2. Create a documented “mode adaptations” ownership rule for print/forced-colors: either keep all mode overrides in `base.css` and list owned selectors, or colocate them beside components like the `.ui-tab` note already argues for (`css/base.css:230`, `css/disclosure.css:55`, `css/base.css:259`).
3. Extract repeated micro-patterns into stable utility classes or custom properties: metadata label (`css/report.css:614`, `css/workbench.css:157`), meta separator (`css/report.css:117`, `css/site.css:351`), link underline (`css/content.css:92`, `css/primitives.css:491`), and details animation (`css/disclosure.css:164`, `css/tree.css:108`).
4. Add a lightweight duplicate-declaration report to CI in advisory mode, scoped to identical blocks across files and allowlisted intentional cases (`package.json:116`, `package.json:191`, `scripts/check-component-matrix.mjs:1`).
5. Add a selector co-ownership map generated from `css/*.css`, excluding comments and explicit mode selectors, so split ownership like table/prose/report and carousel/disclosure/overlay is reviewed intentionally (`css/table.css:6`, `css/content.css:260`, `css/report.css:367`, `css/overlay.css:203`).
6. Prefer full vocabulary for new cross-cutting leaves unless the abbreviation is already public: `.ui-sel*` and `.ui-src*` are stable now, but future leaves should match file names unless docs justify otherwise (`css/selection.css:10`, `css/sources.css:11`, `docs/stability.md:88`, `docs/stability.md:90`).

## Notable observations
- I found no orphan CSS file in the authored graph; the repo has an explicit guard for that case and `node scripts/check-dist.mjs` passed read-only (`scripts/check-dist.mjs:22`, `scripts/check-dist.mjs:33`, `scripts/check-dist.mjs:55`).
- Some overlap is deliberately documented and should not be treated as accidental duplication: app metric/empty-state aliases moved to primitives with permanent aliases (`css/app.css:264`, `css/primitives.css:164`, `css/primitives.css:759`), and crosshair deliberately scopes `.ui-readout` so it does not restyle standalone dot readouts (`css/dots.css:477`, `css/crosshair.css:98`).
- The source/build split is stronger than the source/cohesion split: packaging is generated and checked (`docs/package-contract.md:312`, `scripts/build-dist.mjs:121`), but maintainability debt lives in the hand-authored leaves (`css/primitives.css:853`, `css/report.css:782`, `css/dots.css:666`).
